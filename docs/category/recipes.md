# Documentation Page

URL: https://portdeveloper.github.io/copy-page-button-showcase/docs/category/recipes

[## 📄️Private docs (no AI links)

If your documentation is internal-only or otherwise unreachable from the public internet, the Open in ChatGPT / Claude / Perplexity / Gemini actions won't work — those services can't fetch your URLs.](/copy-page-button-showcase/docs/recipes/private-docs)
[## 📄️Swizzled placement (custom location)

Auto-injection drops the button into the table-of-contents area on desktop. When that doesn't fit your theme — for example you want the button above the "Contents" heading, in a custom header, or alongside an "Edit this page" link — disable injection and render the public React component yourself.](/copy-page-button-showcase/docs/recipes/swizzled-placement)
[## 📄️Markdown URL routes

Enable generateMarkdownRoutes: true and the plugin emits a plain-markdown URL for every documentation page at build time.](/copy-page-button-showcase/docs/recipes/markdown-routes)
[## 📄️MCP server integration

If your documentation site exposes a Model Context Protocol server (or you want to point readers at one you maintain), the plugin can add MCP-related actions to the dropdown:](/copy-page-button-showcase/docs/recipes/mcp-integration)
[## 📄️Custom styling

The button ships with sensible defaults that inherit from the Docusaurus theme. When you need it to look exactly a certain way — match a brand color, position absolutely, hide the chevron — use customStyles or className hooks.](/copy-page-button-showcase/docs/recipes/custom-styling)
